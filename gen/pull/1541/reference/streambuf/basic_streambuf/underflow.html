概要

入力部分列の領域を消費し切った際の処理。

効果

デフォルトの動作はTraits::eof()を返すのみ。（この関数をオーバーライドすることで、関数内で改めてsetg()を呼んで別の入力部分列を指し直す、のような処理を追加できる。）

戻り値

Traits::eof()。

備考

入力部分列の領域を消費し切った状態でsgetc()を呼ぶと、underflow()が呼ばれる。

例

#include <iostream>
#include <streambuf>

// streambufを継承したクラス
class dummy_buf : public std::streambuf {
    char space_[10] = {'A', 'B'};
public:
    dummy_buf(void) {
        // 入力列に配列を設定
        setg(space_, space_, space_ + 2);
    }
protected:
    int_type underflow(void) override {
        std::cout << "underflow" << std::endl;
        // ベースクラスのunderflow()を呼ぶ
        return std::streambuf::underflow();
    }
};

int main() {
    dummy_buf buf{};
    std::cout << buf.sgetc() << std::endl;  // 1文字目の’A'を読み取る
    buf.sbumpc();  // 1文字進める
    buf.sbumpc();  // 1文字進める
    std::cout << buf.sgetc() << std::endl;  // 文字の読み取りに失敗。underflow()が呼ばれ、Traits::eof()が返る
}
